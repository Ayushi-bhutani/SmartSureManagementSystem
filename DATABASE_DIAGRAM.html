🎯 Key Database Design Decisions

1. Database per Service

Each microservice owns its database, ensuring loose coupling and independent scalability. Cross-service references use UserId and PolicyId.

2. GUID Primary Keys

All tables use GUID (uniqueidentifier) as primary keys for distributed system compatibility and avoiding ID collisions.

3. Soft Delete Pattern

No IsDeleted columns - data is permanently deleted or archived. Status fields track lifecycle (Active, Cancelled, Expired).

4. Audit Trail

CreatedAt, UpdatedAt timestamps on all major tables. ClaimStatusHistory tracks all status changes with user attribution.

5. Decimal Precision

All monetary values use decimal(18,2) for precise financial calculations without floating-point errors.

6. Unique Constraints

Email, RegistrationNumber, TransactionReference, Code have unique constraints to prevent duplicates.
